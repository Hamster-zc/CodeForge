# 项目目标

CodeForge 是一个轻量级 Coding Agent Orchestration Harness。

它不是代码生成器，而是一个负责协调多个 Coding Agent 的运行框架。

核心理念：

- 不同 agent 承担不同职责
- 通过 artifact 文件进行上下文传递
- 通过 workflow 控制生命周期
- 通过 policy 控制 agent 路由
- 保留 human-in-the-loop
- 减少单一 agent 的 context 爆炸

v0.1 的目标：

实现一个最小可用闭环：

Task
 ↓
Architect
 ↓
Human approval
 ↓
Implementer
 ↓
Reviewer
 ↓
Verifier
 ↓
Done


不要实现复杂的 multi-agent communication。
不要实现自动训练、向量数据库、长期 memory。
不要过度设计。

# 技术约束

使用 Python 实现。

入口：

codeforge run <task.md>


CodeForge 通过 subprocess 调用已有 CLI：

1. Codex CLI

用于：
- architect
- reviewer
- verifier
- high risk implementation


2. Claude Code CLI

用于：
- low risk implementation

Claude Code 内部可能连接 DeepSeek API。

CodeForge 不直接调用模型 API。

# 核心设计原则


## 1. Role 与 Executor 分离

不要绑定：

junior = DeepSeek
senior = Codex


角色代表职责：

architect:
负责理解项目并设计方案


implementer:
负责代码修改


reviewer:
负责发现问题


verifier:
负责判断任务是否完成


Executor 代表工具：

Codex CLI

Claude Code CLI


未来可以替换。


---

# 2. Artifact-based Communication


Agent 不通过聊天传递信息。

所有重要信息通过文件。


每个任务应该拥有独立目录：

tasks/

└── task_xxx/

    ├── task.md
    ├── state.json

    └── artifacts/

        ├── architecture.md
        ├── architecture.json

        ├── review.md
        ├── review.json

        ├── verification.md
        └── verification.json


Markdown:
给人类阅读。


JSON:
给程序读取。


---

# 3. Context Management


不要让所有 agent 读取整个 repo。


Architect:

输入：

- task
- repository structure
- README
- project metadata


输出：

architecture artifact。


Implementer:

输入：

- task
- architecture artifact
- relevant files


Reviewer:

输入：

- task
- architecture artifact
- git diff


Verifier:

输入：

- acceptance criteria
- test result
- review result


实现一个简单 context.py：

负责准备不同 role 所需上下文。

# 5. Routing


实现简单 router.py。


不要让 LLM 自由决定所有事情。


采用：

Python policy + LLM decision


v0.1 可以简单实现：

policies.yaml:

定义：

low risk:
claude_code


high risk:
codex


例如：

简单文件修改：
claude_code


涉及：

- architecture
- database
- core module

codex


如果无法判断：

默认 codex。

# 8. CLI


最终用户体验：


codeforge run task.md


执行：

1. 创建 task workspace
2. 调 architect
3. 保存 architecture artifact
4. 请求 human approval
5. 调 implementer
6. 调 reviewer
7. 根据 review 结果决定：

   - 简单问题:
       implementer 修复

   - 复杂问题:
       使用更高能力 executor


8. 调 verifier
9. 输出最终结果
